# Release flow

End-to-end checklist for shipping a new Klaudio Panels version.
**Skipping any step here freezes Homebrew users on the previous
version** — the cask bump is non-negotiable.

This doc was extracted from v1.6.4 (2026-05-15) after a release
where `bun tauri build` defaulted to `aarch64-only` and produced a
DMG the existing cask URL pattern couldn't fetch. The
`--target universal-apple-darwin` step below is what fixed it.

Assumes:

- Working dir: `/Users/willywg/proyectos/pet-projects/claude-desktop` (repo `willywg/klaudio-panels`).
- Homebrew tap clone at `~/proyectos/pet-projects/homebrew-klaudio-panels`.
- Both `aarch64-apple-darwin` and `x86_64-apple-darwin` Rust targets installed (`rustup target list --installed`).
- `gh` authenticated against `willywg` for both repos.

## 0. Pre-flight

The PR closing the issue is already merged to `main` and the working
tree is clean. Pull main:

```bash
git checkout main && git pull origin main
```

## 1. Bump versions (4 places — Cargo.lock auto)

| File | Field |
| --- | --- |
| `package.json` | `"version": "X.Y.Z"` |
| `src-tauri/Cargo.toml` | `version = "X.Y.Z"` |
| `src-tauri/tauri.conf.json` | `"version": "X.Y.Z"` |
| `src-tauri/Cargo.lock` | `version = "X.Y.Z"` under `name = "klaudio-panels"` — regenerated by `cargo check` |

After editing the three manual ones:

```bash
cd src-tauri && cargo check && cd ..
```

Cargo bumps the lock file automatically. Verify:

```bash
grep -n -A1 '^name = "klaudio-panels"' src-tauri/Cargo.lock | head -5
```

## 2. CHANGELOG entry

Add a new section at the top of `CHANGELOG.md` immediately after the
header paragraph, before the previous version's entry. Style: Keep a
Changelog, ISO date. Sections used in this repo: **Added / Changed /
Fixed / Removed / Tracked work** (the last one links PRP + PR + issue).

Template (replace `X.Y.Z`, the date, the punchline, and the links):

```markdown
## [X.Y.Z] — YYYY-MM-DD

### Added | Changed | Fixed
- **Punchline** ([#NN](https://github.com/willywg/klaudio-panels/issues/NN)).
  Body — what changed, why, how it works, any trade-offs the reader
  should know.

### Tracked work
- PRP: [`PRPs/0NN--slug.md`](PRPs/0NN--slug.md)
- PR: [#NN](https://github.com/willywg/klaudio-panels/pull/NN)
- Issue: [#NN](https://github.com/willywg/klaudio-panels/issues/NN)
```

## 3. Commit + tag + push

```bash
git add CHANGELOG.md package.json src-tauri/Cargo.lock src-tauri/Cargo.toml src-tauri/tauri.conf.json
git commit -m "chore(release): vX.Y.Z — <punchline>"
git tag vX.Y.Z
git push origin main
git push origin vX.Y.Z
```

The tag is what GitHub uses to anchor the release page to a
specific commit; the cask URL also embeds `vX.Y.Z`. Don't skip it.

## 4. Build universal DMG

```bash
bun tauri build --target universal-apple-darwin
```

⚠️ **`--target universal-apple-darwin` is mandatory.** Without it Tauri
builds a host-arch DMG (e.g. `Klaudio Panels_X.Y.Z_aarch64.dmg`) and
the cask URL pattern `Klaudio.Panels_#{version}_universal.dmg` won't
resolve. Intel Macs would get `404 Not Found` on upgrade.

Output:

```
src-tauri/target/universal-apple-darwin/release/bundle/dmg/Klaudio Panels_X.Y.Z_universal.dmg
```

Build is 4–8 min on a fresh tree, 1–3 min incremental. Run in the
background so you can prep the release notes in parallel.

## 5. SHA256 + rename to cask filename

The build produces `Klaudio Panels_…dmg` (with a **space**). The cask
URL expects `Klaudio.Panels_…dmg` (with a **dot**). Copy with the
right name so the upload lands on the URL the cask will fetch:

```bash
DMG="src-tauri/target/universal-apple-darwin/release/bundle/dmg/Klaudio Panels_X.Y.Z_universal.dmg"
SHA=$(shasum -a 256 "$DMG" | awk '{print $1}')
echo "SHA256: $SHA"
cp "$DMG" "/tmp/Klaudio.Panels_X.Y.Z_universal.dmg"
```

The SHA is byte-identical for both filenames; the rename is purely
about the URL path GitHub serves.

Keep `$SHA` around — step 7 needs it.

## 6. GitHub release (with DMG attached)

```bash
gh release create vX.Y.Z \
  --repo willywg/klaudio-panels \
  --title "vX.Y.Z — <punchline>" \
  --notes "<release body — see template below>" \
  "/tmp/Klaudio.Panels_X.Y.Z_universal.dmg"
```

Release notes template (mirrors CHANGELOG entry but with full URLs
since GitHub doesn't resolve repo-relative paths in release bodies):

```markdown
## Added | Changed | Fixed
- **Punchline** ([#NN](https://github.com/willywg/klaudio-panels/issues/NN)).
  Body.

## Tracked
- PRP: [`PRPs/0NN--slug.md`](https://github.com/willywg/klaudio-panels/blob/vX.Y.Z/PRPs/0NN--slug.md)
- PR: [#NN](https://github.com/willywg/klaudio-panels/pull/NN)
- Issue: [#NN](https://github.com/willywg/klaudio-panels/issues/NN)

## Install / upgrade

```bash
brew update && brew upgrade --cask klaudio-panels
```

Or grab the DMG below.
```

Note the `vX.Y.Z` in the PRP link path — it pins to the tag, not
`main`, so old release pages stay accurate after the file rotates
or is renamed in a future PRP.

## 7. Bump the Homebrew cask (non-negotiable)

The cask repo is a separate clone at
`~/proyectos/pet-projects/homebrew-klaudio-panels`. Two fields change:

```ruby
cask "klaudio-panels" do
  version "X.Y.Z"
  sha256 "<$SHA from step 5>"
  # ... rest stays the same
end
```

```bash
cd ~/proyectos/pet-projects/homebrew-klaudio-panels
# Edit Casks/klaudio-panels.rb
git add Casks/klaudio-panels.rb
git commit -m "klaudio-panels X.Y.Z — <short punchline>"
git push origin main
```

The cask repo's `main` is the tap; users on `brew update && brew
upgrade --cask klaudio-panels` get the new version the moment this
push lands. **No release notes, no tag, no PR** — single commit on
`main`.

## 8. Smoke test

From a clean shell:

```bash
brew update
brew upgrade --cask klaudio-panels
# or for a fresh install: brew install --cask klaudio-panels
```

Open the app, verify the version in About (or the title bar) reads
`X.Y.Z`. macOS may show a Gatekeeper warning on first launch — the
cask's `caveats` block documents the right-click → Open workaround.

## Anti-patterns to avoid

- ❌ Building without `--target universal-apple-darwin` — Intel users break.
- ❌ Uploading the DMG with the space in its filename — cask URL 404s.
- ❌ Skipping the cask bump — `brew upgrade` reports "already up to date" and Homebrew users get nothing.
- ❌ Amending the release commit after the tag is pushed — the tag points to the pre-amend SHA, the release page becomes inconsistent. Just make a follow-up `chore(release): X.Y.Z+1` instead.
- ❌ Hand-editing `Cargo.lock` instead of running `cargo check` — easy to drift from the other manifests.
- ❌ Mixing release notes from multiple PRPs into one release without each having its own changelog section.
